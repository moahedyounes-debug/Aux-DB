## خطة تطوير الداشبورد (4 مراحل)

### المرحلة 1 — الهوية والتصميم
- تصميم لوجو AUX جديد احترافي (SVG أصلي، ليس النص الحالي في `AuxLogo.tsx`) بلونين: فاتح وداكن.
- وضع اللوجو في:
  - صفحة تسجيل الدخول (`index.html`) بحجم كبير مع تحسين شكل الفورم.
  - القائمة الجانبية (`DashboardLayout.tsx`) بشكل مضغوط.
- تحسينات بصرية شاملة: تحسين البطاقات (KpiCard/ChartCard)، مسافات، ظلال ناعمة، حركات دخول، وحدة أيقونات ثابتة.
- الحفاظ على الألوان الحالية في `src/styles.css` مع إضافة gradients و shadows للأزرار الرئيسية.

### المرحلة 2 — نظام Access والأدوار
البنية: كل شيء يبقى في Google Sheets (كما طلبت).

**توسيع تبويب Access في الشيت** ليصبح بالأعمدة التالية:
```
Email | ASC | Branch | Role | Pages (comma-separated) | Admin | Parts | Call Center
```

- `Role`: نص حر (Super Admin / Manager / Call Center / أو أي اسم جديد يكتبه الأدمن).
- `Pages`: قائمة مسارات الصفحات المسموحة (مثل `/kpis,/call-center`)، أو `all` للكل.
- إذا كانت الخانة فارغة، يعود للسلوك القديم (كل الصفحات مفلترة حسب ASC/Branch).

**صفحة Access الجديدة (للأدمن فقط):**
- جدول تفاعلي بكل المستخدمين مع تعديل مباشر (inline edit).
- زر "إضافة مستخدم جديد" و"حذف".
- عمود Pages يعرض كـ **checkboxes متعددة** لكل صفحة في `NAV_PAGES` — كل صفحة عليها tag يمكن إزالته أو إضافته.
- زر "إنشاء دور جديد" — يفتح modal يحدد اسم الدور والصفحات الافتراضية.
- الحفظ يكتب مباشرة إلى Google Sheet عبر server function جديدة `access.functions.ts` باستخدام PUT على gateway الشيتس (بصلاحيات الكاتب المتوفرة في الكونكتور).

**تطبيق الصلاحيات:**
- تحديث `useAccess` ليقرأ `Pages` وإرجاع `allowedPages: string[]`.
- تحديث `DashboardLayout` للسايدبار: يعرض فقط الصفحات الموجودة في `allowedPages` (أو الكل لو `all`).
- كل route جديد يتحقق من الصلاحية ويعيد التوجيه إذا لم تكن الصفحة مسموحة.

### المرحلة 3 — محرر الداتا + معادلات KPI
**محرر الشركات و OBM:**
- صفحة جديدة `/data-editor` (للأدمن) — تبويبات: Companies, OBM, KPI Formulas.
- كل تبويب: جدول + أزرار (إضافة/تعديل/حذف/إخفاء) — الإخفاء = عمود `Hidden = Yes` في الشيت.
- الكتابة إلى Google Sheets عبر server functions تستخدم مفتاح الكونكتور الحالي.

**محرر معادلات KPI:**
- تبويب في نفس الصفحة.
- كل معادلة: `KPI Name | Formula | Format (number/%/currency) | Hidden`
- صيغة نصية بسيطة مثل: `=(closed_tickets/total_tickets)*100`
- قائمة بالمتغيرات المتاحة (مقروءة من عناوين أعمدة الشيت الرئيسي) تظهر تحت مربع الكتابة.
- التقييم عبر مقيّم آمن (لن نستخدم `eval`؛ سنستخدم مكتبة `expr-eval` — تُضاف عبر bun add).
- المعادلات المحفوظة تُعرض تلقائياً في صفحة KPI Scorecard.

### المرحلة 4 — صفحة Customer Lookup
- مسار جديد `/customer-lookup`.
- حقل بحث: رقم جوال / اسم / رقم تذكرة.
- تجميع من كل الشيتات المتاحة (Maintenance, Calls, Parts, Complaints, WhatsApp) وعرض:
  - بيانات العميل الأساسية.
  - Timeline زمنية بكل تفاعلاته (تذاكر، مكالمات، رسائل، شكاوى، قطع غيار).
  - إحصائيات سريعة (عدد الزيارات، آخر تفاعل، الحالة).

### الأمان والأداء
- الأدمن فقط يشوف صفحات Access و Data Editor.
- caching عبر React Query موجود.
- كل عمليات الكتابة تمر عبر server functions (لن يصل مفتاح الشيت للمتصفح).
